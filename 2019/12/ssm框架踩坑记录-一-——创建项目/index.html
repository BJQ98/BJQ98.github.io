ssm框架踩坑记录(一)——项目创建
发表于 2019-12-16| 更新于 2019-12-27|技术
字数总计: 873|阅读时长: 2 分钟| 阅读量:

鉴于本人用一个框架之前总会各种踩坑，ssm也不例外。在网上多次下载他人demo无法运行起来以及看他人blog一步步走还是会出现诸多问题，于是决定记录一下一个最简单的ssm小项目搭建的过程。本篇许多内容都是来自慕课网的一个免费课程 ，只不过我的目的只是为了搭建起一个小项目。

项目创建

准备工具：idea、jdk1.8、Maven3.5.4

这里工具和环境的安装就不赘述了，直接打开我们的idea，选择新建项目:

按照我图中的选择，点击next:

其中groupid是域名的反写，artifactid是项目的名称，点击next即可:

这一步有一个注意点，因为idea默认会从网上下载资源加载项目，往往会导致项目创建时卡死在网络io中，少则十几分钟，多则数小时（可能），所以为了采取我们本地的maven设置，我们在这个阶段点击左下的加号，设置参数

archetypeCatalog=internal

接着一路next直至finish即可，创建完成后的目录是这样子的:

至此，我们的项目创建完成了。

项目相关文件

准备工具：idea

首先在我们创建好的项目中，找到main文件夹，在该目录下创建java和resources两个文件夹，并分别在文件夹上右击，选择Mark Directory as，分别标记为Sources Root和Resources Root，并且在src目录下新建test文件夹，在test文件夹中新建java并且标记为Test Resources Root，用于存放以后的测试目录。完成之后的结果如下图所示：

接下来在我们的蓝色java文件夹中创建四个package，分别新建com.xxx.controller、com.xxx.dao、com.xxx.entity、com.xxx.service，其中com.xxx为你项目域名的反写，至于为什么要四次创建都带com.xxx，因为在创建的过程中，还没在包中新建java类的话，idea会默认在当前目录下创建，比如你创建好com.xxx.controller之后，直接创建dao，就会出现com.xxx.controller.dao的情况。
然后我们在resources创建mapper和spring两个文件夹，至此，项目创建的工作基本算是完成了:
